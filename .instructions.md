# Code Generation Rules for Ronin

When generating code for any Ronin project, follow these principles:

## Core Code Quality Rules
- **Keep code simple** — avoid unnecessary complexity; favor readability over cleverness
- **Keep code clean** — follow naming conventions, consistent formatting, proper documentation
- **Think analytical and goal oriented** — understand requirements first, implement with clear purpose
- **Use design patterns and SOLID principles** — apply proven patterns for maintainability

## Development Workflow
- **Test code after chunks of steps are done** — verify each logical unit works before proceeding
- **Ensure integration frontend-backend works after chunks of steps are done** — test API contracts and data flow
- **Run tests** — execute full test suite after changes; verify no regressions
